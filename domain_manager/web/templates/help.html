🏠 O systému

Domain Manager (Dominator) je webová administrační konzola pro správu školní IT infrastruktury postavené na Samba Active Directory. Umožňuje spravovat počítače, uživatele, skupiny, DHCP, DNS filtrování, síťové topologie a automatizaci — vše z jednoho místa bez nutnosti přímého přístupu na server.

Technologie

  • Backend: Python / FastAPI, SQLAlchemy, SQLite / PostgreSQL
  • Frontend: Jinja2 šablony, HTMX (bez Node.js, bez npm)
  • AD backend: Samba 4 (ldap3 klient)
  • Všechny JS/CSS závislosti jsou lokální — systém funguje i bez přístupu k internetu

Demo režim

Pokud se zobrazuje žlutý banner DEMO REŽIM, všechny write operace jsou blokovány a pracujete s ukázkovými daty. Demo je určeno pro testování UI bez reálného AD serveru. Spustíte jej pomocí DM_DEMO=1 dm web start.

📊 Přehled

Úvodní stránka po přihlášení. Zobrazuje aktuální stav infrastruktury na jednom místě:

  • Statistiky AD — počet počítačů, uživatelů, skupin
  • Zdraví serverů — ping, dostupnost portů (DC1, DC2, Pi-hole…)
  • Nedávné události — posledních N záznamů z audit logu
  • Aktivní DHCP pronájmy — přehled přiřazených IP adres
Přehled se automaticky neobnovuje. Stiskněte F5 pro načtení aktuálních dat.

🖥 Počítače

Seznam všech počítačových účtů v Active Directory. Umožňuje základní operace s objekty AD.

Co lze dělat

  • Přidat počítač — vytvoří objekt v AD (neinstaluje OS)
  • Přesunout do OU — změna organizační jednotky v AD
  • Zakázat/Povolit účet — blokuje přihlášení počítače do domény
  • Smazat — odstraní objekt z AD
  • Blokovat internet — přeřadí klienta do Pi-hole skupiny dm-blokováno

Ikony stavů

  • 🟢 Online — počítač odpovídá na ping
  • 🔴 Offline — nedostupný
  • ⚫ Neznámo — ping nebyl proveden

Import z CSV

Tlačítko ↑ Import CSV v záhlaví stránky umožňuje hromadné vytvoření nebo aktualizaci počítačů.

Povinný sloupec: hostname. Záhlaví lze psát česky i anglicky. Existující záznam (stejný hostname) se aktualizuje, nový se vytvoří.

hostname,mac,ip,department,location,building,floor,room,asset_tag,serial_number,manufacturer,model,os_family,primary_user,warranty_until,purchase_date
pc-ucebna-01,aa:bb:cc:dd:ee:01,192.168.10.101,Učebny,Přízemí,A,0,U01,INV-001,SN12345,Dell,OptiPlex 7090,windows,novak,2027-06-30,2024-01-15
pc-ucebna-02,aa:bb:cc:dd:ee:02,192.168.10.102,Učebny,Přízemí,A,0,U01,,,,,,windows,,
  • Datum ve formátu YYYY-MM-DD, DD.MM.YYYY nebo DD/MM/YYYY
  • Zaškrtnutí Vytvořit i v AD zavolá samba-tool computer create pro nové záznamy (vyžaduje běh na DC)
Blokování internetu přes Pi-hole funguje pouze pokud klienti dotazují Pi-hole přímo (ne přes DC jako forwarder). Pro garantované blokování použijte Ansible s pravidly nftables.

👤 Uživatelé

Správa uživatelských účtů v Active Directory.

Dostupné operace

  • Vytvořit uživatele — jméno, příjmení, přihlašovací jméno (UPN), heslo, OU
  • Změnit heslo — reset bez znalosti stávajícího hesla (vyžaduje admin práva)
  • Zakázat/Povolit účet — rychlé zablokování bez mazání
  • Přidat do skupiny — hromadné přiřazení
  • Smazat — nevratná operace, buďte opatrní

Import z CSV

Tlačítko ↑ Import CSV vedle tlačítka "+ Nový uživatel". Povinné sloupce: username, first_name, last_name.

username,first_name,last_name,email,phone,department,job_title,title,manager
novakj,Jan,Novák,novak@skola.cz,+420123456789,Učitelé,Učitel,Mgr.,rednik
svobodap,Pavel,Svoboda,svoboda@skola.cz,,IT,Správce sítě,Bc.,novakj
  • Záhlaví lze psát česky (jméno, příjmení, oddělení…) nebo anglicky
  • Existující uživatelé (stejné username) se aktualizují
  • Zaškrtnutí Vytvořit i v AD volá samba-tool user create; AD heslo lze zadat v poli vedle
Smazání uživatele v AD je nevratné. Zvažte nejprve zakázání účtu.

👥 Skupiny

AD skupiny slouží k hromadnému přiřazování práv, GPO filtrování a organizaci uživatelů.

  • Vytvořit skupinu — tlačítko + Nová skupina AD, zadejte název, typ a popis; vytvoří se v DB i v AD
  • Smazat skupinu — odstraní z DB i z AD (potvrzovací dialog); odebrání skupiny neodstraní její členy

Import z CSV

Tlačítko ↑ Import CSV. Záhlaví je volitelné, pořadí sloupců: name, kind, description.

name,kind,description
Ucitele,user,Skupina učitelů
PC-Ucebna1,computer,Počítače v učebně 1
PC-Ucebna2,computer,Počítače v učebně 2
  • kind: computer nebo user (lze psát česky: počítače, uživatelé)
  • Zaškrtnutí Synchronizovat s AD vytvoří skupiny i přes samba-tool group add
  • Existující skupiny (stejný název) jsou přeskočeny
Skupiny se používají ve Politikách GPO pro cílené nasazení nastavení.

🌐 DHCP

Správa DHCP serveru — přehled rozsahů, aktivních pronájmů a statických rezervací.

Záložky

  • Pronájmy — aktuálně přiřazené IP adresy s MAC adresou a hostname
  • Rezervace — statické přiřazení IP → MAC (klient vždy dostane stejnou IP)
  • Rozsahy — DHCP pool: od–do, maska, gateway, DNS

Vytvoření rezervace

  1. V tabulce Aktivní lease klikněte + Rezervovat — IP, MAC a hostname se přenesou automaticky
  2. Nebo vyplňte formulář Přidat rezervaci ručně
  3. Od příštího obnovení pronájmu bude klient dostávat tuto IP
DHCP je čteno přímo ze Samba/ISC DHCP démona přes SSH nebo lokální soubor. Změny se projeví při dalším obnovení pronájmu klientem.

🕳 Pi-hole

DNS content filtering — blokování reklam, trackerů a malwaru na úrovni DNS pro celou síť.

Skupiny a klienti

Pi-hole v6 umožňuje per-klient blokování přes skupiny. Klienti ve skupině dm-blokováno mají blokován přístup k internetu přes DNS. Klienti v default mají standardní filtrování.

Blokující seznamy (adlisty)

  • Aktuální seznamy — tabulka aktivních adlistů s počtem domén a datem aktualizace
  • Předvolby — kurátorský výběr populárních seznamů; zaškrtněte a klikněte Přidat vybrané
  • Vlastní URL — přidejte libovolný adlist z internetu
  • Zapnout/Vypnout — každý seznam lze dočasně deaktivovat bez smazání

Aktualizace gravity

Po přidání nových seznamů je nutné spustit Aktualizovat gravity — Pi-hole stáhne všechny adlisty a zkompiluje je do lokální databáze. Operace trvá 1–5 minut.

Per-klient blokování funguje pouze pokud klienti dotazují Pi-hole přímo. Pokud klienti jdou přes DC (Samba DNS forwarder), Pi-hole vidí všechny dotazy jako od DC a per-klient nastavení nemá efekt.

⚙️ Ansible

Automatizace konfigurace klientů a serverů přes SSH bez nutnosti agentů.

Spuštění playbooku

  1. Vyberte playbook ze seznamu
  2. Zadejte limit (volitelné) — název hosta, skupiny nebo all
  3. Klikněte Spustit — spustí se job a přesměruje na jeho výstup
  4. Výstup se automaticky aktualizuje dokud job nedoběhne

Editor playbooků

Zabudovaný editor s YAML zvýrazňováním syntaxe a knihovnou snippetů:

  • Ctrl+/ — zakomentovat/odkomentovat řádek
  • Ctrl+F — hledání v kódu
  • Tab — odsazení (2 mezery)
  • Panel vpravo — snippety seskupené po kategoriích; klik vloží šablonu na pozici kurzoru

Inventář

Inventář je generován automaticky z dat v AD. Skupiny Ansible odpovídají OU v AD.

Playbook musí mít příponu .yml nebo .yaml. Název nesmí obsahovat lomítka ani ..

📋 Politiky GPO

Správa Group Policy Objects (GPO) v Samba AD.

  • Vytvořit politiku — GPO objekt v AD s volitelným popisem
  • Propojit s OU — politika se aplikuje na objekty v dané OU
  • Upravit nastavení — otevře editor nastavení (pokud je dostupný RSAT nebo samba-tool)
  • Vynutit / Zakázat dědičnost — prioritizace nebo blokování GPO
Změny GPO se projeví při dalším obnovení skupinových politik na klientovi (gpupdate /force) nebo po přihlášení.

🗺 Plánky sítě

Vizuální editor pro zakreslení rozmístění zařízení do půdorysů budovy.

Struktura

Plánky jsou hierarchické — místnost může být pod patrem, patro pod budovou. Každý plánek má název a volitelný obrázek (SVG, PNG, JPG).

Práce se zařízením

  • Přidat zařízení — přetáhněte ikonu ze sloupce vpravo na plánek
  • Přesunout — klikněte a držte na ikoně, poté táhněte
  • Info panel — krátký klik levým tlačítkem zobrazí detail zařízení (IP, stav)
  • Kontextové menu — pravý klik na ikonu nabídne akce (detail, odebrat z plánu)
  • Odebrat ze scény — zařízení se vrátí do sloupce nepřiřazených; z AD se nesmaže
Pozice se ukládají okamžitě po puštění tlačítka myši. Obrázek plánku se nahrává přes formulář editace.

📡 Monitorování sítě

Přehled síťových zařízení (switche, routery, AP) monitorovaných přes SNMP a ping.

  • Přidat zařízení — zadejte IP, SNMP community string a typ
  • Stav — zelená = dostupné, červená = nedostupné, žlutá = degradované
  • SNMP data — uptime, název, verze firmware, zatížení portů (pokud zařízení podporuje)
  • Topologie — schéma propojení zobrazené jako graf
SNMP v2c je výchozí. Pro SNMP v3 je potřeba zadat auth/priv hesla v konfiguraci zařízení.

🔐 Certifikáty

Přehled TLS certifikátů na serverech a sledování jejich platnosti.

  • Přidat certifikát — zadejte hostname:port nebo nahrajte PEM soubor
  • Platnost — barevná indikace: zelená (30+ dní), žlutá (7–30 dní), červená (<7 dní)
  • Upozornění — systém může zasílat notifikace před vypršením (pokud jsou nastaveny)

💾 Zálohy

Správa záloh konfigurace a dat Domain Manageru i Samba AD.

  • Vytvořit zálohu — spustí zálohovací skript a uloží archiv
  • Stáhnout — zálohu si můžete stáhnout přímo do prohlížeče
  • Obnovit — import zálohy (pouze konfigurace DM, nikoli AD databáze)
  • Plánované zálohy — nastavení přes cron v konfiguraci serveru
Obnova zálohy přepíše aktuální konfiguraci. Zálohu AD databáze (samba-tool domain backup) provádějte odděleně.

💚 Zdraví systému

Kontrolní panel dostupnosti všech součástí infrastruktury.

  • DC1, DC2 — dostupnost Samba AD (LDAP port 389, Kerberos 88)
  • Pi-hole — DNS port 53, webové API
  • DHCP — port 67
  • Domain Manager — vlastní stav (vždy OK pokud vidíte tuto stránku)

Každá kontrola zobrazuje latenci a čas posledního ověření. Stránka se neobnovuje automaticky — použijte tlačítko Obnovit.

📝 Audit log

Chronologický záznam všech akcí provedených v Domain Manageru.

  • Každá write operace (vytvoření, úprava, smazání) zanechá záznam s časovým razítkem, uživatelem, typem akce a detaily
  • Log je uložen v databázi Domain Manageru (nikoli v AD)
  • Filtrování — podle uživatele, akce nebo časového rozsahu
  • Export — záznamy lze exportovat jako CSV
Audit log sleduje pouze akce provedené přes webové rozhraní. Přímé změny v AD (přes RSAT, samba-tool) se zde nezobrazí.

⌨️ CLI příkazy

Domain Manager lze ovládat z příkazové řádky přes nástroj dm.

Správa webového serveru

dm web start           # spustí server na popředí
dm web start --daemon  # spustí jako daemon na pozadí
dm web stop            # zastaví server
dm web restart         # restart
dm web status          # zobrazí stav (PID, port, log)

Demo režim

DM_DEMO=1 dm web start --daemon

Správa konfigurace

dm config show         # zobrazí aktuální konfiguraci
dm config set key val  # nastaví konfigurační hodnotu

Databáze

dm db migrate          # spustí migrace databáze
dm db seed             # naplní demo daty

Ansible

dm ansible run playbook.yml         # spustí playbook
dm ansible run playbook.yml --limit dc1
Nápověda k libovolnému příkazu: dm <příkaz> --help
